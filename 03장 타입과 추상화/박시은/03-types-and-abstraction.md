# 03. 타입과 추상화

## 추상화

- 어떤 **양상, 세부 사항, 규칙**을 더 명확하게 이해하기 위해  
  특정 절차나 물체를 의도적으로 **생략하거나 강조함으로써 복잡도를 극복하는 방법**

- 구체적인 사물들 간의 **공통점은 취하고**,  
  **차이점은 버리는 일반화**를 통해 대상을 단순하게 만드는 것

- 중요한 부분을 강조하기 위해  
  **불필요한 세부 사항을 제거**함으로써 단순하게 만드는 것

- 추상화의 목적은  
  **복잡함을 이해하기 쉬운 수준으로 단순화**하는 데 있다.

- 추상화의 **수준, 이익, 가치는 목적에 의존적**이다.

```
public class Door {

    private boolean locked;
    private boolean opened;

    public void open() {
        if (locked) {
            throw new IllegalStateException("문이 잠겨 있습니다.");
        }
        this.opened = true;
    }

    public void close() {
        this.opened = false;
    }

    public void lock() {
        if (opened) {
            throw new IllegalStateException("열린 문은 잠글 수 없습니다.");
        }
        this.locked = true;
    }
}
```
## 개념

- **공통점을 기반으로 객체들을 묶기 위한 그릇**
- 개념을 이용하면 객체를 **여러 그룹으로 분류**할 수 있다.


## 개념의 세 가지 관점

- **심볼(Symbol)**  
  : 개념을 가리키는 **간결한 이름이나 명칭**

- **내연(Intension)**  
  : 개념의 **정의를 나타내며**,  
    내연의 의미를 이용해 객체가 **개념에 속하는지 여부를 판단**

- **외연(Extension)**  
  : 해당 개념에 **속하는 모든 객체의 집합(Set)**

## 데이터 타입(type)
- 메모리 안에 저장된 데이터의 종류를 분류하는 데 사용하는 메모리 집합에 관한 메타데이터

## 타입 시스템의 목적
- 메모리 안의 모든 데이터가 비트열로 보임으로써 야기되는 혼란을 방지하는 것.
- 데이터가 잘못 사용되지 않도록 제약사향을 부과하는 것.

## 객체와 타입
- 어떤 객체가 어떤 타입에 속하는지를 결정하는 것은 객체가 수행하는 행동이다.
- 객체의 내부적인 표현은 외부로부터 철저하게 감춰진다.
- 동일한 책임(= 동일한 메시지 수신)을 수행하는 일련의 객체는 동일한 타입에 속한다.
- 객체를 분류하는 기준은 타입이다.
- 클래스는 타입을 구현할 수 있는 한가지 방법이다.

## 일반화/특수화 관계
- 일반화/특수화 관계를 결정하는 것은 객체의 행동이다.
- 일반적인 타입은 특수한 타입보다 외연의 크기는 더 크고 행동의 수는 더 적다.

## 슈퍼타입과 서브타입
- 슈퍼타입 : 좀 더 일반적인 타입
- 서브타입 : 좀 더 특수한 타입
- 서브타입은 슈퍼타입을 대체할 수 있어야 한다.

## 동적 모델과 정적 모델
- 동적 모델 : 객체들의 동작과 흐름을 설명 (ex. 시퀀스 다이어그램)
- 정적 모델 : 시스템의 구조를 설명 (ex. 클래스 다이어그램)
