# 01. 협력하는 객체들의 공동체

## 객체란
- 객체는 현실 세계에 존재하는 사물에 대한 **추상화**이다.
- **상태(State)** 와 **행동(Behavior)** 을 함께 지닌 실체이다.
- 다른 객체가 **무엇(What)** 을 수행하는지는 알 수 있지만,  
  **어떻게(How)** 수행하는지에 대해서는 알 수 없다.

## 역할, 책임, 협력
- 객체 지향 시스템은 **역할(Role)**, **책임(Responsibility)**, **협력(Collaboration)** 을 중심으로 구성된다.

## 협력의 성공
- 특정한 역할을 맡은 각 객체가  
  **얼마나 요청을 성실히 이행하는가**에 따라 협력의 성공 여부가 결정된다.

## 역할의 의미
- 역할은 **대체 가능성**을 의미한다.
- 동일한 역할을 수행할 수 있다면 객체는 서로 교체 가능하다.

## 다형성 (Polymorphism)
- 동일한 요청에 대해  
  **서로 다른 방식으로 응답할 수 있는 능력**

## 책임이 불분명한 객체의 문제
- 책임이 불분명한 객체는  
  애플리케이션의 **전체 구조 역시 불분명하게 만든다.**

## 전지전능한 객체 (God Object)
- 외부의 도움을 무시한 채  
  모든 것을 스스로 처리하려는 객체
- 내부 복잡도가 지나치게 높아져  
  **유지보수와 확장이 어려워진다.**

## 메서드 (Method)
- 메서드는 객체가 **메시지를 처리하는 방법**이다.

## 프로토타입 기반의 객체 지향
- 클래스 없이 객체를 생성한다.
- 객체가 다른 객체의 속성과 메서드를 **상속받는 방식**이다.
