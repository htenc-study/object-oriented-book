# 01장 협력하는 객체들의 공동체

객체지향이란 클래스들의 구조가 아니라, **역할을 가진 객체들이 메시지를 주고받으며 협력하는 과정**이다.

## 협력하는 사람들
1. 협력에 참여하는 구성원들은 요청과 응답을 통해 협력한다.
- 요청: 손님이 캐셔에게 커피를 주문한다.
- 응답: 캐셔는 손님에게 완성된 커피를 손님에게 전달한다.

2. 협력하는 과정 속에서 특정 역할을 부여받는다.
- 커피를 주문하는 요청에서 요청자는 손님, 응답자는 캐셔라는 역할을 부여받는다.

3. 각 역할에는 책임이 존재한다.
- 손님은 커피를 주문할 책임이 있다.
- 캐셔는 손님에게 커피가 완성되었음을 알릴 책임이 있다.

## 역할, 책임, 협력
* 여러 객체가 동일한 역할을 수행할 수 있다.
    - 다형성 (상속 또는 인터페이스 구현을 통한 실현)
* 역할은 대체가능성을 의미한다.
    - 어떤 객체를 사용하더라도 동일한 책임을 수행한다는 보장을 제공한다.
* 각 객체는 책임을 수행하는 방법을 자율적으로 선택할 수 있다.
    - 메시지와 메서드의 분리(캡슐화)
* 하나의 객체가 동시에 여러 역할을 수행할 수 있다.
    - 역할은 객체보다 먼저 정의되며, 객체는 역할을 수행하기 위해 선택된다.


## 협력 속에 사는 객체
1. 객체는 충분히 협력적이어야 한다.
- 다른 객체에게 도움을 요청할 수 있어야한다.
2. 객체는 충분히 자율적이어야 한다.
- 요청에 대해 스스로 판단하고 응답한다.

* 객체는 상태와 행동을 함께 지닌 자율적인 객체이다.


1. 객체들은 **메시지**를 통해 요청 및 응답한다.
2. 객체가 수신된 메시지를 처리하는 방법을 **메서드**라고 한다.

* 메시지와 메서드를 분리하는 것은 객체의 자율성을 높이는 핵심 메커니즘이고, 이것은 캡슐화라는 개념과도 깊이 관련되어 있다.
- 캡슐화: 객체가 자기 책임을 수행하는 방법을 외부로부터 숨기는 것
- 객체와 관련된 책임(메서드)을 객체에 넣는 것


## 객체지향의 본질
* 객체지향이란 시스템을 상호작용하는 자율적인 객체들의 공동체로 바라보고 객체를 이용해 시스템을 분할하는 방법이다.
* 자율적인 객체란 상태와 행위를 함께 지니며 스스로 자기 자신을 책임지는 객체를 의미한다.
* 객체는 시스템의 행위를 구현하기 위해 다른 객체와 협력한다. 각 객체는 협력 내에서 정해진 역할을 수행하며 역할은 관련된 책임의 집합니다.
* 객체는 다른 객체와 협력하기 위해 메시지를 전송하고, 메시지를 수신한 객체는 메시지를 처리하는 데 적합한 메서드를 자율적으로 선택한다.

* 객체지향의 핵심은 클래스가 아닌, 적절한 책임을 수행하는 역할 간의 유연하고 견고한 협력관계를 구축하는 것.

* 클래스는 단지 협력에 참여하는 객체를 만드는데 필요한 구현 메커니즘일 뿐이다.
